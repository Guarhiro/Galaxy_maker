# 🌌 A-plan 星図ギャラリー（Galaxy Maker）

夜空に浮かぶ星をクリックすると、キャラクターや作品の紹介ポップアップが開く——そんな
**インタラクティブな「星図（スターマップ）」ギャラリー**をブラウザ上で作れるツールです。

作った星図は、画像も音楽も**1枚のHTMLファイルに丸ごと埋め込んで書き出せる**ので、
そのファイルをアップロードするだけで誰でも公開できます。サーバーもデータベースも不要です。

> **リポジトリ:** <https://github.com/Guarhiro/Galaxy_maker>
> **技術:** React 19 + Vite 6（モダンなフロントエンド構成）

---

## ✨ できること

- 🌟 **星を最大20個まで配置** — 名前・色・サイズ・位置を自由に設定
- 🖼️ **星ごとに画像とテキスト** — キャラ立ち絵、シーン画像、紹介文、リンク先作品を登録
- 🌌 **背景画像の差し替え** — 好きな星空・宇宙画像をアップロードまたはURL指定
- 🎵 **BGM対応** — 全体BGMに加えて、星ごとに個別BGMも設定可能
- 👀 **公開表示プレビュー** — 編集画面と「公開後の見た目」をワンクリックで切り替え
- 💾 **自動保存** — 編集内容はブラウザ内（localStorage）に自動保存
- 📦 **公開HTMLの書き出し** — 画像を埋め込んだ自己完結型HTMLを1ファイルで出力

## 📦 はじめる前に（必要なもの）

このツールを動かすには **Node.js** というソフトが必要です（1回入れれば以降ずっと使えます）。

### Node.js を入れる

1. [Node.js公式サイト](https://nodejs.org/ja) を開く
2. **「LTS」** と書かれた方（推奨版）のボタンをクリックしてダウンロード
3. ダウンロードしたインストーラーを開き、案内に従って「次へ」を押していくだけ

> 💡 バージョンは **18以上**であればOKです（このプロジェクトは Node.js 24 で動作確認済み）。

### 入っているか確認する

「ターミナル」（Windowsなら「コマンドプロンプト」または「PowerShell」、Macなら「ターミナル」アプリ）
を開いて、次の2行をそれぞれ入力し、Enterキーを押してください。

```bash
node -v
npm -v
```

`v24.15.0` や `11.12.1` のように**数字が表示されれば成功**です。
「コマンドが見つかりません」と出る場合は、Node.js のインストールがまだ完了していません。

## 🚀 導入手順（ステップ・バイ・ステップ）

ターミナルに、下のコマンドを**上から順に1行ずつ**入力してEnterでOKです。

### ① プロジェクトをダウンロードする

```bash
git clone https://github.com/Guarhiro/Galaxy_maker.git
```

> 💡 `git` が入っていない場合は、GitHubのページ右上の緑色の **「Code」▾ → 「Download ZIP」** から
> ダウンロードして解凍してもOKです。

### ② プロジェクトのフォルダに移動する

```bash
cd Galaxy_maker
```

> 💡 `cd` は「このフォルダの中に入る」という意味のコマンドです。
> 以降の作業はすべて、このフォルダの中で行います。

### ③ 必要な部品をインストールする

```bash
npm install
```

このツールが使っている部品（React など）を自動でダウンロードします。
回線にもよりますが、**初回は1〜3分ほど**かかります。`node_modules` というフォルダが
作られたら成功です（このフォルダは触らなくてOKです）。

### ④ 開発サーバーを起動する

```bash
npm run dev
```

成功すると、ターミナルに次のような表示が出ます👇

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://127.0.0.1:5173/
```

### ⑤ ブラウザで開く

表示された **`http://127.0.0.1:5173/`**（または `http://localhost:5173/`）を
ブラウザのアドレスバーに入力して開いてください。星図ギャラリーの編集画面が表示されます🌌

> 💡 **止めたいとき:** ターミナルで `Ctrl + C` を押すとサーバーが停止します。
> もう一度使うときは、再び `npm run dev` を実行するだけです。

## 🖋️ 基本的な使い方

| やりたいこと | 操作 |
| --- | --- |
| 星を追加する | 画面上部の **「星を追加」** ボタン（最大20個まで） |
| 星の内容を編集する | 左の一覧から星を選び、右側のパネルで名前・画像・紹介文などを入力 |
| 背景を変える | 左パネルの **「背景」** で画像URLを入力 or 「画像を選択」でアップロード |
| BGMを設定する | 上部の **「BGM GitHub URL」** に音声ファイルのURLを入力 |
| 公開後の見た目を確認 | 上部の **「公開表示」** ボタンで切り替え（戻すときは「編集表示」） |
| 内容を保存する | **自動保存**されます（ブラウザを閉じても消えません） |

> 💡 画像やBGMは「URL指定」が基本です。GitHub等にアップした画像の **raw URL**
> （`https://raw.githubusercontent.com/...`）を貼り付けると確実です。

### 現在の公開BGM

GitHub Pages（`docs/` 公開）では、次の音源をリポジトリ内に配置しています。

| 用途 | ファイル |
| --- | --- |
| 一覧画面BGM | `docs/assets/audio/gallery.mp3` |
| エテルニア・ステラリアBGM | `docs/assets/audio/ginga-wa-warawa-no-teatime.mp3` |

編集画面では `public/assets/audio/` 側の同名ファイルを参照します。作品詳細を開くと作品BGMへ切り替わり、閉じると一覧BGMの止まった秒数から再開します。

## 📦 公開HTMLを書き出す（公開のしかた）

作った星図を世界に公開するには、画面上部の **「公開HTML」** ボタンを押します。

- 画像などを埋め込んだ **`a-plan-star-gallery-public.html`** が1ファイルで保存されます
- このHTMLファイル単体で動くので、**ホスティングサービスにアップするだけ**で公開できます
  （GitHub Pages、Netlify、レンタルサーバー等、どこでもOK）

> 📝 別途、`npm run build` を実行すると `dist/` フォルダに本番用ビルドも出力できます。
> 「公開HTML」ボタンは、その手軽版（1ファイル完結）だと考えてください。

## ❓ うまくいかないとき

| 症状 | 対処 |
| --- | --- |
| `node` や `npm` が「見つかりません」 | Node.js のインストールをやり直す → [公式サイト](https://nodejs.org/ja) |
| `npm install` でエラーになる | リポジトリ内の `.npmrc` にある `cache=...` の行を削除してから再実行してください（作者環境のキャッシュ先が書かれているため） |
| ポートが使用中だと出る | 既に別の `npm run dev` が動いていないか確認。`Ctrl + C` で止めてから再実行 |
| 画面が真っ白 | ブラウザを再読み込み（リロード）。それでもダメなら一度サーバーを止めて `npm run dev` |
| 変更が反映されない | このツールは保存と同時に自動更新（HMR）されます。されない場合はブラウザをリロード |
